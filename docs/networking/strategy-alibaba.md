### 2.2 具体挖掘路径

**路径1：Shopee → 阿里国际 → 阿里云**

Shopee深圳研发中心的前同事中，必然有人跳去了阿里国际（Lazada深圳团队）。阿里国际和阿里云虽属不同BU，但同在深圳阿里中心办公，内部转介绍成本低。

操作步骤：
1. 翻看微信/脉脉通讯录中的Shopee前同事
2. 查看他们的当前公司 → 筛选出在"阿里巴巴"的
3. 优先联系：
   - 在阿里国际的（深圳同城，最容易约见面）
   - 在阿里云的（直接相关）
   - 在阿里任何部门的（至少可以了解内部文化和转介绍）

话术：
```
XX好！好久不见，我是之前跟你一起在Shopee做XX的张三。

我离开Shopee后去字节待了两年，现在在腾讯做云控制台前端架构。

最近看到阿里云在招做低代码平台的前端方向，跟我过往的
JSON Schema动态表单项目经验很对口。

你在阿里也待了一段时间了，想跟你请教几个问题——
阿里云深圳团队的办公地点和氛围怎么样？
内部转介绍（cross-BU referral）方便吗？

不着急，你方便的时候回我就好。什么时候有空也可以约个饭！
```

**路径2：字节 → 阿里**

字节和阿里之间的人才流动频繁。你字节的前同事网络中，大概率有人去了阿里。

操作：
1. 翻看微信中字节前同事的当前公司
2. 如在阿里 → 走A层/B层联系策略
3. 如不在但关系好 → 问"你有没有认识的阿里深圳团队的朋友"

**路径3：腾讯 → 阿里云**

腾讯云和阿里云是直接竞品，人才双向流动虽不频繁但存在。你的CSIG同事中：
- 可能认识从阿里云跳来腾讯的人（反向了解阿里云内部情况）
- 可能有CSIG同事的朋友/前同事在阿里云

操作：
1. 在团队内部私下问："有人知道阿里云低代码团队的情况吗？"
2. 通过CSIG的行业交流活动找到阿里云的人

**路径4：开源社区 → 阿里前端核心圈**

这是最不依赖现有人脉、但需要耐心的路径。

阿里的前端开源生态包括：
- Ant Design / AntV（蚂蚁系，但与阿里云有交叉）
- Fusion Design / ice / rax（阿里系）
- 阿里前端技术博客 / 掘金专栏

操作：
1. 找到Fusion Design / ice 的GitHub仓库
2. 查看contributors中标注为"阿里云"的开发者
3. 通过GitHub issues/PR/讨论区互动
4. 从技术讨论自然过渡到："我对阿里云低代码平台很感兴趣，方便加个微信吗？"

## 三、脉脉/知乎冷启动战术

### 3.1 脉脉精准搜索

**第一轮：定位目标人物**

搜索关键词组合：
```
"阿里云 低代码平台" "阿里云 aPaaS"
"阿里云 前端架构师"
"阿里云智能 前端"
"阿里云 深圳 前端"
"阿里云 可视化搭建"
```

**关键筛选维度**：
- 职级：P7+（才有能力做内推/提供有质量的情报）
- 活跃度：近30天有动态（说明活跃，会回复）
- 内容质量：有技术分享（可成为你联系的切入点）

**第二轮：建立存在感**

1. 关注5-10个目标人物
2. 对他们的技术动态进行有深度评论（非"学习了""厉害"）
3. 你自己发1-2篇技术动态：
   - 《从JSON Schema动态表单到低代码平台：3个关键架构决策》
   - 《微前端架构在云控制台的工程实践》

**第三轮：私信联系**

```
[对方称呼]你好，关注你一段时间了，
你在[具体文章/动态]里提到的[具体内容]对我启发很大。

我目前在腾讯云做前端架构，之前在字节做过动态表单引擎，
刚好看到阿里云在招低代码平台方向的前端技术专家，
跟我过往的项目经验非常对口。

不知道方不方便加个微信，请教你几个关于
你们团队技术方向的问题？不打扰，10分钟语音就行。
```

**为什么这样写**：
- 先建立共鸣（具体提及对方内容）
- 清晰说明自己是谁（不是陌生人）
- 提到对口项目（技术共鸣）
- 低压请求（10分钟语音 = 低投入）

### 3.2 知乎专家连接

**目标KOL识别**：
- 搜索"阿里云 低代码" "阿里前端"找到阿里工程师的知乎账号
- 找到后：关注 → 对其回答点赞 → 在评论区提有深度的问题

**有深度的评论示例**：
```
非常好的分享。想问一个架构层面的问题：
阿里云低代码平台的Schema驱动渲染层，
在动态表单校验和联动逻辑的实现上，
是走的编译时方案（Schema→Vue/React组件静态化）
还是运行时方案（全量Schema动态解析）？
两种方案的性能边界大致在什么量级？
```

**效果**：展现技术深度 → 吸引对方注意 → 后续私信连接更有基础。

## 五、内推 vs 自主投递的决策矩阵

### 5.1 阿里内推的四种质量等级

| 内推人级别 | 简历权重 | 能获取的信息 | 获取难度 |
|-----------|---------|-------------|---------|
| **P8+ 直属团队成员** | 极高，HR必定联系且可能优先安排 | 团队氛围、HC详情、面试重点 | 最高 |
| **P7+ 目标部门同事** | 高，简历优先查看 | 团队概况、面试风格 | 中高 |
| **P6/P7 其他部门** | 中，简历标记"内推" | 公司基本情况 | 中 |
| **网上找的通用内推码** | 基本等同于自主投递 | 几乎无 | 极低 |

### 5.2 决策

**目标**：争取P8+或P7+直属团队成员内推。

**如果2-3周内找不到高质量内推人**：
- 退而求其次，通过阿里国际的Shopee前同事做跨BU内推
- 跨BU内推在阿里体系里仍有价值（简历标记为内推来源）
- 同时在脉脉/猎聘上自主投递，双线并行

### 7.2 微信/语音 Informational Call 问题清单

建立微信联系后，约15-20分钟语音时问：

**技术方向**（展现专业度）：
1. 阿里云低代码平台的aPaaS定位是面向开发者还是业务人员？
2. 平台的Schema驱动渲染层架构是什么？运行时解析还是编译时方案？
3. 团队目前在意的技术挑战是什么？

**团队氛围**（评估适配性）：
4. 团队规模和前端/后端比例？
5. 近期团队有没有组织调整？对未来方向有影响吗？
6. 一般几点下班？（问具体时间，不是"加班多吗"）

**面试建议**（如果到投递阶段）：
7. 这个岗位面试最看重什么？有没有特别关注的考察点？
8. 团队对P7/P8的期望差异是什么？
