## 一、活水转岗全景图

### 1.1 活水是什么

腾讯内部转岗机制（"活水计划"）允许在岗员工申请其他BG/BU的岗位，无需离职再入职。是内部职业发展的核心通道。

### 1.2 你的活水资格

| 条件 | 要求 | 你的状态 | 
|------|------|---------|
| 在岗时间 | ≥3个月 | 满足（3年+ ✅） |
| 绩效 | 最近一次非"待改进" | 满足（推断 ✅） |
| 目标岗位 | 公开发布的正式HC | TEG/WXG有目标岗位 ✅ |
| 主管沟通 | 建议提前沟通 | 策略见下文 |

### 1.3 两种活水模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **保密活水** | 不通知当前主管，完成新部门面试后再告知 | 不想过早暴露意图时（推荐） |
| **公开活水** | 先告知主管，主管支持后正式转岗 | 与主管关系好、主管愿意放人时 |

> 策略建议：采用**保密活水**模式。拿到TEG/WXG的offer后再与CSIG主管沟通。制度上活水不经过当前主管审批，但提前告知是职场礼仪。拿到offer后沟通，既保护自己，也留有体面。

## 四、活水申请流程与时间线

### 4.1 标准流程

```
阶段1: 寻找岗位      阶段2: 人脉预热      阶段3: 申请投递      阶段4: 面试         阶段5: 转岗交接
[1-3天]              [3-7天]              [1天]               [1-2周]            [2-4周]
    │                   │                    │                   │                  │
    ├─ HR助手活水板块    ├─ 目标团队社交联系    ├─ 活水系统提交       ├─ 3轮技术面       ├─ 原团队交接
    ├─ 内推系统活水岗    ├─ 了解团队氛围       ├─ 上传简历+项目     ├─ 1轮HR面         ├─ 确定last day
    └─ KM/TGIDEAS发布    └─ 确认HC有效性       └─ 等待HR联系        └─ 收到offer       └─ 入职新BG
```

### 4.2 关键时间节点

| 阶段 | 最佳时机 | 说明 |
|------|---------|------|
| 提交申请 | 绩效评估结束后 **或** 年终奖到账后 | 避免影响当次绩效评核 |
| 面试 | 工作日午休或傍晚 | 不影响现团队正常排期 |
| 告知主管 | 拿到offer后 | 保密活水模式下，避免提前暴露 |
| 离职交接 | 利用30天通知期 | 活水通常要求30天交接期 |

> **2026年5月的时间建议**：如果去年年终奖已到账，现在（5月）就是最佳活水时机。如果Q2绩效评估在6月，建议6月中旬评估结束后立即提交。

### 4.3 活水面试与普通社招的区别

| 维度 | 活水 | 普通社招 |
|------|------|---------|
| 面试轮次 | 通常2-3轮 | 3轮+ |
| 简历筛选 | 内部员工优先查看 | 与外部候选人竞争 |
| 反馈速度 | 快（1-2周内） | 较慢（2-4周） |
| 薪资谈判 | 通常平移或小幅提升，年终连续计算 | 重新谈package |
| HC竞争 | 活水专用HC或共享HC | 社招HC |
| 离职交接 | 内部调动，无需离职 | 需离职再入职 |

## 五、联系话术库

### 5.1 初次联系（企业内部微信/TGIDEAS）

**场景A：KM/TGIDEAS 文章互动后联系**

```
[称呼]老师你好，我是CSIG的前端工程师张三。

最近在KM上读到你们关于[具体技术主题]的文章/分享，
里面的[具体技术细节]给我启发很大。

我们CSIG云控制台这边也在做类似的[技术方向]，
有几个技术选型上的困惑想请教一下。

如果你方便的话，想约个咖啡（食堂也行）简单聊聊，
大概20分钟就好。不方便也没关系，我整理一下问题发你也OK。

谢谢！
```

**为什么有效**：
- 以具体技术话题切入，非直接求职
- 尊重对方时间（20分钟限定）
- 具体共鸣点（非泛泛"文章很好"）
- low-pressure（给了"不方便也没关系"的退路）

```
[同事名]跟我说你跟他在[场合/项目]上合作过，

最近在考虑通过活水到TEG（关注你们[具体团队]），
```

## 六、风险识别与应对

### 6.1 核心风险

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| 保密活水被提前泄漏 | 中 | 现主管关系紧张，留或不留都尴尬 | 只告诉最信任的1-2人；避免在公司电脑上查看活水岗位；面试安排在午休或flex时间 |
| TEG/WXG HC冻结或收回 | 低 | 活水流程中断 | 申请前通过内部联系人确认HC真实有效 |
| 现主管拒绝放人 | 低 | 有offer但走不了 | 制度上活水无需现主管批准；遇到强硬不放人时，可找活水HR协调 |
| 转岗后发现团队不合 | 低 | 6个月内不能再次活水 | 面试时反向考察团队，问清楚workload和工作方式 |
| 年终奖受影响 | 中 | 跨BG转岗可能被原BG"优化"年终 | 选择年终奖发放后启动活水；或拿到offer时跟HR确认年终连续性 |

### 6.2 活水黑话解读

| 行话 | 真实含义 |
|------|---------|
| "团队节奏快" | 加班多 |
| "需要owner意识" | 周末也可能需要响应 |
| "技术挑战大" | 遗留系统多/hack多 |
| "需要强势推动" | 跨部门扯皮多 |
| "小而美的团队" | 资源配置可能不足 |

## 七、目标团队反向评估清单

联系到TEG/WXG内部人员后，务必确认以下问题：

### 必问清单

- [ ] 团队目前多少人？前后端比例如何？
- [ ] 前端团队的技术栈真的用了React 18还是停留在16？
- [ ] 最近一年有没有组织调整/人员流动？为什么？
- [ ] 每天普遍几点下班？（不要问"加班多吗"，问具体时间）
- [ ] 最近一个迭代的deadline压力大吗？
- [ ] 团队现在最头疼的技术问题是什么？
- [ ] 活水过来的同事适应如何？

### 建议问（建立信任后）

- [ ] 领导的管理风格偏放权还是偏细节？
- [ ] 团队晋升到T10难度大吗？最近一个晋升的是什么时候？
- [ ] 年终奖最近一年的平均水平？

## 八、活水成功后90天融入计划

| 阶段 | 时间 | 关键动作 |
|------|------|---------|
| Onboarding | 第1-2周 | 熟悉团队架构、代码库、开发流程；找到一位buddy |
| Early win | 第3-4周 | 交付一个小但有可见性的改进（比如一个CI优化、一个组件bug修复） |
| 架构理解 | 第5-8周 | 深入理解团队技术栈的架构决策和trade-off；开始参与技术评审 |
| 独立贡献 | 第9-12周 | 承担独立模块；在技术讨论中提出有深度的建议 |
| 建立影响 | 90天后 | 持续技术分享；参与技术委员会；争取Tech Lead角色 |
